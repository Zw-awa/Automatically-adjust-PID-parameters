## 📦 这是什么工具？

简单说：这是一个**用AI帮你调PID参数**的工具。

**PID是什么？**
- 一种控制算法，让机器（比如电机、机器人）按照你想要的方式运动
- 有三个参数：Kp、Ki、Kd（不用管具体含义，AI会帮你调）

**这个工具能做什么？**
- 自动分析你的控制系统数据
- 用AI（DeepSeek大模型）给出调参建议
- 帮你找到最优的PID参数

## 🚀 5步快速上手

### 第1步：安装Python（如果还没装）

**Windows用户看这里**：
1. 打开浏览器，访问：https://www.python.org/downloads/
2. 点击黄色的"Download Python"按钮
3. 下载完成后，双击安装
4. **重要**：安装时一定要勾选"Add Python to PATH"（如下图）
   ```
   [✓] Add Python to PATH
   ```
5. 点击"Install Now"，等待安装完成

**验证安装成功**：
1. 按 `Win + R`，输入 `cmd`，回车
2. 输入：`python --version`
3. 应该显示：`Python 3.8.x` 或更高版本

### 第2步：下载项目

**方法A（推荐）**：直接下载ZIP
1. 下载项目ZIP文件
2. 解压到桌面（或其他方便的位置）
3. 你会看到一个 `pid_tuner_project` 文件夹

**方法B**：使用Git（如果你知道Git）
```bash
git clone <项目地址>
```

### 第3步：安装所需软件包

1. 打开命令提示符（cmd）
2. 进入项目文件夹：
   ```bash
   cd Desktop\pid_tuner_project
   ```
   （如果你的项目不在桌面，改成实际路径）

3. 安装软件包：
   ```bash
   pip install -r requirements.txt
   ```

**如果安装很慢**，用这个命令：
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 第4步：获取API密钥（关键步骤！）

这个工具需要调用DeepSeek的AI服务，所以需要一个API密钥。

**获取步骤**：
1. 访问：https://platform.deepseek.com/
2. 点击"Sign up"注册账号（用邮箱注册）
3. 登录后，点击右上角头像 → "API Keys"
4. 点击"Create new API key"
5. 给密钥起个名字，比如"PID调参工具"
6. **复制生成的密钥**（长串字符，像`sk-xxxxxxxxxxxx`）

**重要**：这个密钥就像密码，不要分享给别人！

### 第5步：配置项目

1. 在项目文件夹里，找到 `config.json` 文件
2. 用记事本（或任何文本编辑器）打开它
3. 找到这一部分：
   ```json
   "llm": {
     "api_key": "your-deepseek-api-key",
   ```
4. 把 `"your-deepseek-api-key"` 替换成你刚才复制的真实密钥
   ```json
   "llm": {
     "api_key": "sk-你复制的真实密钥",
   ```
5. 保存文件

## 🎮 第一次运行：仿真模式

现在可以测试了！输入这个命令：

```bash
python main.py simulate --loop speed --iterations 2
```

**命令解释**：
- `simulate`：仿真模式（不用真实硬件）
- `--loop speed`：测试速度控制环
- `--iterations 2`：运行2次调参

**你会看到**：
```
============================================================
  PID Auto-Tuner - Simulation Mode
  Loop: speed
  Iterations: 2
============================================================

----------------------------------------
  Simulation iteration #1
  PID: Kp=1.0000 Ki=0.1000 Kd=0.0500
----------------------------------------
Performance Metrics:
  Overshoot: 15.2%      ← 超调量（越小越好）
  Settling Time: 0.42s  ← 调节时间（越短越好）
  Steady-State Error: 0.8% ← 稳态误差（越小越好）

Consulting LLM...      ← AI正在思考

----------------------------------------
  LLM Analysis:
  Confidence: 85%
  Reason: 超调量偏高，建议减小Kp，增加Kd来抑制振荡
  
  Parameter Changes:
    Kp: 1.000000 → 0.800000 (-20.0%)
    Ki: 0.100000 → 0.150000 (+50.0%)
    Kd: 0.050000 → 0.030000 (-40.0%)
```

**恭喜！** 🎉 你已经成功运行了AI PID调参系统！

## 📚 理解输出结果

### 关键指标

| 英文名 | 中文名 | 什么意思 | 理想值 |
|--------|--------|----------|--------|
| **Overshoot** | 超调量 | 响应超过目标值的程度 | < 5% |
| **Settling Time** | 调节时间 | 达到稳定所需时间 | < 1秒 |
| **Steady-State Error** | 稳态误差 | 稳定后的误差 | < 2% |

**简单理解**：
- 超调量大了 → 系统"冲过头"了
- 调节时间长了 → 系统反应"慢"
- 稳态误差大了 → 系统"不准"

### AI的建议

- **Confidence（置信度）**：AI对自己的建议有多自信（85%就是85%把握）
- **Reason（理由）**：AI为什么这样建议
- **Parameter Changes（参数变化）**：Kp、Ki、Kd怎么调整

## 🔧 三种使用模式

### 1. 🎮 仿真模式（推荐新手）
**特点**：不用硬件，纯软件模拟
```bash
# 基本命令
python main.py simulate --loop speed --iterations 3

# 测试不同控制环
python main.py simulate --loop steering --iterations 2  # 转向环
python main.py simulate --loop position --iterations 2  # 位置环
```

### 2. 📁 离线模式（有数据时用）
**特点**：分析已有的CSV数据文件
```bash
# 分析示例数据
python main.py offline --file data/raw/example_speed_data.csv --loop speed

# 分析你自己的数据（把文件放到data/raw/文件夹）
python main.py offline --file data/raw/你的数据.csv --loop speed
```

**数据格式要求**：
```
timestamp,target,actual,error,output
0.0000,100.0,0.0,100.0,500.0
0.0100,100.0,4.85,95.15,480.75
0.0200,100.0,9.42,90.58,462.9
```

### 3. 🔌 在线模式（有硬件时用）
**特点**：连接真实设备，实时调参
```bash
# Windows（串口通常是COM3、COM4等）
python main.py online --port COM3 --loop speed --interval 10

# 询问是否应用新参数时，按y确认，按n跳过
```

## 🆘 常见问题解决

### Q1：提示"ModuleNotFoundError"
```
ModuleNotFoundError: No module named 'openai'
```
**解决**：重新安装依赖
```bash
pip install -r requirements.txt
```

### Q2：提示"Authentication error"
```
ERROR: LLM call failed: Authentication error
```
**解决**：检查 `config.json` 中的 `api_key` 是否正确

### Q3：命令不识别
```
'python' 不是内部或外部命令
```
**解决**：Python没安装或没添加到PATH，重新安装并勾选"Add Python to PATH"

### Q4：想从头开始
**解决**：删除 `outputs/` 文件夹和 `config.json` 中的历史记录
